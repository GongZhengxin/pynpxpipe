## 3. BHV2 文件结构（已知）

```
┌─────────────────────────────────────────┐
│ IndexPosition (magic)                   │  ← uint64 LE 偏移值
├─────────────────────────────────────────┤
│ FileInfo (struct)                       │  ← machinefmt + encoding
├─────────────────────────────────────────┤
│ Trial1, Trial2, ... Trial11             │  ← trial 数据
│ MLConfig                                │  ← session 配置（MonkeyLogic 写入）
│ （其他未知变量，由 who() 枚举）           │
├─────────────────────────────────────────┤
│ FileIndex (at offset from IndexPosition)│  ← cell[N,3]: name/start/end
└─────────────────────────────────────────┘
```

变量通用格式：
```
uint64: name_length
char[name_length]: variable_name
uint64: type_length
char[type_length]: type_string
uint64: ndim
uint64[ndim]: size_array
<type-specific payload>
```

## 4. MATLAB 导出脚本设计

### 4.1 脚本位置与入口

```
docs/ground_truth/export_ground_truth.m   ← 主脚本（用户手动运行）
```

```matlab
bhv2_file = 'F:\#Datasets\demo_rawdata\241026_MaoDan_YJ_WordLOC.bhv2';
output_dir = 'tests\fixtures\bhv2_ground_truth';
export_bhv2_ground_truth(bhv2_file, output_dir);
```

### 4.2 导出变量

| 变量 | 说明 | 输出文件 |
|------|------|---------|
| `Trial1` ~ `Trial11` | 完整 trial struct | `trial_01.json` ~ `trial_11.json` |
| `MLConfig` | session 级别配置 | `ml_config.json` |
| `FileInfo` | 编码/字节序 struct | `file_info.json` |
| `FileIndex` | 变量索引 cell array | `file_index.json` |
| 元信息 | 文件哈希、变量列表等 | `index.json` |

若 `who()` 返回额外变量，记录到 `index.json` 的 `extra_variables` 字段（不强制导出，避免未知大型变量阻塞）。

### 4.3 类型序列化规则

| MATLAB 类型 | JSON 表示 | 说明 |
|------------|----------|------|
| `double/single` scalar | `number`（`%.17g`） | IEEE 754 exact round-trip |
| 数值数组（任意 dtype） | `{"_t":"nd","dt":"float64","sh":[490,2],"d":[...]}` | Fortran order 展平，shape 保留 |
| `char` | `"string"` | 直接字符串，反斜杠 JSON escape |
| `logical` | `true` / `false` | |
| `struct` | `{"field":...}` | 递归展开 |
| `cell` | `[...]` | 递归，元素类型可混合 |
| `containers.Map` | `{"_t":"map","keys":[...],"values":[...]}` | |
| `function_handle` | `{"_t":"fh","v":"func2str 结果"}` | |
| `NaN` | `"__NaN__"` | JSON 不支持 NaN |
| `Inf` / `-Inf` | `"__Inf__"` / `"__-Inf__"` | |
| `table/timetable/timeseries` | `{"_t":"unsupported","class":"table"}` | 占位，不阻断脚本 |
| `ml*` 前缀类型 | 视为 `struct` | 兼容旧版 BHV2，与 mlbhv2.m:304 一致 |

**精度**：`%.17g` 保证 IEEE 754 double 的 exact round-trip（Python `float(s)` 还原后 bit pattern 相同）。

**Fortran order**：数组以列优先顺序展平存入 `d`。Python 端还原：`np.array(d, dtype=dt).reshape(sh, order='F')`。

### 4.4 index.json 格式

```json
{
  "source_file": "241026_MaoDan_YJ_WordLOC.bhv2",
  "md5": "...",
  "trial_count": 11,
  "exported_variables": ["Trial1", "...", "Trial11", "MLConfig", "FileInfo", "FileIndex"],
  "extra_variables": [],
  "exported_at": "2025-xx-xx HH:MM:SS",
  "matlab_version": "R20xx"
}
```

## 5. 格式文档结构（B1.1）

**文件**：`docs/specs/bhv2_binary_format.md`

```
1. 文件总体布局（附 hex dump 前 64 字节标注）
2. Magic / IndexPosition：字节偏移表（0-7: uint64 name_len=13, 8-20: 'IndexPosition', 21+: body）
3. FileInfo 变量：machinefmt/encoding 字段
4. 变量通用格式：6 字段逐字节规格表
5. Payload 规格（对照 mlbhv2.m read_variable 各 case）：
   - 数值类型：fread dtype → numpy dtype 映射表
   - struct：uint64 nfield → nfield 个递归字段
   - cell：prod(size) 个递归元素（无 nfield）
   - containers.Map：两个连续变量 keys + values
   - ml* 兼容处理
6. FileIndex 结构：cell[N,3]，三列含义，偏移为 double 需转 int
7. 已验证字节偏移表（来自真实文件 hex dump，标注哪些与 spec 一致）
```

## 6. mlbhv2.m 分析文档结构（B1.2）

**文件**：`docs/specs/bhv2_matlab_analysis.md`

```
1. 文件概览（class mlbhv2 < handle，属性表）
2. open()：三种模式（r/w/a）的分支，read_index 调用时机
3. read_index()：magic 识别 → FileInfo → FileIndex 定位逻辑
4. 三种读取入口对比：read() / read_trial() / who()
5. read_variable()（核心，逐 case 注释）：
   - 数值：fread + reshape Fortran order
   - struct：nfield 先读，递归 nfield 个字段
   - cell：prod(sz) 个元素递归，无 nfield
   - containers.Map：read_variable(keys) + read_variable(values)
   - ml* 前缀：强制转 struct（兼容注释）
   - function_handle：func2str / str2func
   - EOF 异常：mlbhv2:eof identifier
6. 陷阱清单（Python 实现必读）：
   - sz=[1,1] 时 prod=1，仍有 shape，不等于 scalar
   - fread 返回 double，uint64 大偏移超 2^53 会丢精度（文件安全上限约 9PB，实际无影响）
   - FileIndex 偏移存为 double，Python 端需 int() 转换后才能 seek
   - FileInfo 可能不存在（旧版文件），需 fallback 到默认编码 windows-1252
```

## 9. 风险

| 风险 | 缓解 |
|------|------|
| MLConfig 含 `function_handle` 或 `table`，序列化失败 | 用 `{"_t":"unsupported"}` 占位，不让单个变量阻断整个脚本 |
| `who()` 返回未预期的大型变量 | 只强制导出已知变量，未知变量仅记录到 `index.json`，不导出内容 |
| hex 验证发现 `bhv2_reader.md §2` 有错误 | 以 hex 验证结果为准，在 `bhv2_binary_format.md` 中标注差异，同步修正 `bhv2_reader.md §2` |
